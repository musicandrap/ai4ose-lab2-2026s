# AI4OSE实验二
2026.02.27

## 一、引言

当前，AI 对话系统与智能编程辅助技术正以前所未有的速度演进，为操作系统内核研发带来深刻变革——既是效率提升的机遇，也是方法论重构的挑战。部分同学希望在操作系统课程中深入探索 “AI for OS Engineering”（AI4OSE） 的科研与工程实践路径。
为此，BigLab-B 鼓励学生结合 AI Agent 技术、Rust 组件化系统工程方法、现代 OS 内核优化技术，围绕一个真实的类 Linux 内核项目（StarryOS），开展以 AI 协同为核心的创新开发，旨在探索下一代操作系统研发的新范式。

## 二、目标
- 总体目标  
在 AI 协同与轻量级团队协作的支持下，围绕 StarryOS 内核，开展面向功能、性能与安全的系统级工程实践，推动 AI+Rust 在类 Linux 内核开发中的深度应用，并形成可复用、可评估、可共享的技术成果。

- 具体目标
在教师与助教指导下，学生需：
 - 与 AI 工具深度协作，学习基于 `arceos` 相关组件的组件化内核开发方法，学习相关例子，完成5个小练习；
 - 与其他同学保持轻量级、异步式协同，共同研发：基于 Arceos-Based StarryOS（获2025全国大学生OS比赛一等奖，已支持约 200 个 Linux 系统调用，兼容 Alpine Linux 与 X11 图形界面，支持 RISC-V64/LoongArch64/AArch64/x86_64 四种架构，支持ebpf扩展, gcc, make, vim, git, qemu, python3, SQLite, Redis, curl+jq等）；
 - 扩展其功能或提升性能或增强安全性（详见考核方式）


## 三、要求
1. 周学习进展记录  

    每周提交一份 学习进展纪要。将与各类 AI 工具的交互记录（提问、代码生成、调试对话等）按时间顺序归档，统一存入个人 Git 仓库，作为过程性评估依据。

2. 关于附录资源的使用建议  

   附录提供大量“可选资源”，涵盖多个基于 Rust 的类 Linux 内核项目（如 StarryOS、asterinas、DragonOS）及教学实验体系。无需全部掌握，也不必全盘照搬。鼓励借助 AI 快速理解、筛选、借鉴，构建适合自身节奏的开发路径。

3. 技术栈与工程规范  

   基础平台：StarryOS（GitHub）  
   语言：Rust  
   运行环境：QEMU 模拟器 或 物理硬件  
   开发模式：基于 Rust Crate 的组件化架构  
   质量要求：所有新增内核模块或主程序必须通过单元测试与系统测试，并满足独立发布至 crates.io 的要求。

4. 文档规范

   所有文档须使用 Markdown 格式编写，图表应采用纯文本方式绘制（推荐使用 Mermaid 语法）。

5. 开源与迭代  

   开发仓库须托管于指定的 公开 Git 仓库；初版无需完美，倡导 “快速发布、持续迭代”；  公开透明便于教师与社区提供反馈，共同推动改进。

6. AI 工具的使用原则  

   不限制 AI 工具类型，鼓励充分、创造性地与各类 AI 协作；但核心目标是提升自身系统能力与工程素养，而非依赖工具替代思考；
   在学生达到一定阶段后，课程组将视情况提供更高级 AI 工具支持（注：因涉及成本，无法保证全覆盖）。

## 四、考核

给yuchen@tsinghua.edu.cn发电子邮件，标题是[AI4OSE-lab2-2026s passed]-姓名+学校名，邮件内容是学生的仓库链接和其他能代表学生通过了实验的重要信息。

考核指标：


1. 基础实验实践（20%）  
   
   充分利用各类AI工具，完成附录“最新教学实验教程”( https://github.com/rcore-os/tg-arceos-tutorial/ )中所列的5个基本练习（位于 app-helloworld/app-collections/app-msgqueue/app-loadapp/app-runlinuxapp 中），夯实操作系统理论基础并提升实践能力，最终提交一份包含与AI合作的实现过程与学习收获的总结报告。  
   - 与AI合作的实现过程（包括如何与AI交互，碰到的问题/bug、解决过程等）
   - 学习效果评估（包括自己在这个学习过程中知识和能力的提升/下降评价，与本校现有教学实验教程的定量/定性的对比分析）

2. 核心项目开发（40%）  
   
   阅读老师指定相关的学术论文和科研项目，与 AI 协作，并与其他同学异步松耦合协同，共同研发，扩展 StarryOS 的功能/性能/安全，完成如下内容之一，并提交完整设计总结报告。
    * 成功运行至少一个主流 Linux 应用（如 YOLOv8、Syzkaller、Rust Compiler、Nginx、Python、Wayland等）
    * 提升StarryOS性能（I/O,调度,内存管理,文件系统,多核优化等之一）至少50%以上
    * 设计工具/方法，找到并修复StarryOS中5类（涉及并发、内存、安全、语义、正确性的类型，至少2类bug，bug数量10个以上）的Bug
    * 完成老师认可的OS相关新功能

4. 原理掌握与实践能力考核（40%）  
   通过面试评估学生对所做工作的理解深度与实际动手能力。
   学生需要做slide对自己的工作进行总结，并回答老师提出的问题。
 
考核评价（百分制，满分100分）：
得到80分以上：通过
得到90分以上：优秀
我们会通过电子邮件发通过验证信息。
